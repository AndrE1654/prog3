# Завдання: Управління обліковими записами з різними рівнями доступу

**Умова:**  
1. Спроектуйте класи для управління обліковими записами користувачів з різними рівнями доступу до системи.<bd>
Створіть базовий клас User (Користувач) з атрибутами:<bd>
username (ім'я користувача)<bd>
password_hash (хеш пароля)<bd>
is_active (булеве значення, що вказує, чи активований обліковий запис).<bd>
Метод verify_password(password), який приймає пароль та порівнює його з password_hash.<bd>
2. Створіть підкласи, що представляють різні ролі користувачів, наприклад:<bd>
Administrator (Адміністратор), який успадковує від User та може мати додаткові атрибути або методи, пов'язані з адмініструванням системи (наприклад, список дозволів).<bd>
RegularUser (Звичайний користувач), який також успадковує від User та може мати специфічні для звичайних користувачів атрибути (наприклад, остання дата входу).<bd>
GuestUser (Гість), який є підкласом User та може мати обмежені права доступу.<bd>
3. Створіть клас AccessControl (Контроль доступу) з атрибутами:<bd>
users (словник, де ключами є імена користувачів, а значеннями - об'єкти класів користувачів).<bd>
Метод add_user(user), який додає нового користувача до системи.<bd>
Метод authenticate_user(username, password), який перевіряє, чи існує користувач з таким ім'ям та чи правильний введений пароль. Повертає об'єкт користувача у разі успішної аутентифікації, і None у разі невдачі.<bd>
