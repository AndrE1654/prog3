# Завдання: Управління обліковими записами з різними рівнями доступу

**Умова:**  
1. Спроектуйте класи для управління обліковими записами користувачів з різними рівнями доступу до системи.<br>
   - Створіть базовий клас User (Користувач) з атрибутами:<br>
   - username (ім'я користувача)<br>
   - password_hash (хеш пароля)<br>
   - is_active (булеве значення, що вказує, чи активований обліковий запис).<br>
   - Метод verify_password(password), який приймає пароль та порівнює його з password_hash.<br>
2. Створіть підкласи, що представляють різні ролі користувачів, наприклад:<br>
   - Administrator (Адміністратор), який успадковує від User та може мати додаткові атрибути або методи, пов'язані з адмініструванням системи (наприклад, список дозволів).<br>
   - RegularUser (Звичайний користувач), який також успадковує від User та може мати специфічні для звичайних користувачів атрибути (наприклад, остання дата входу).<br>
   - GuestUser (Гість), який є підкласом User та може мати обмежені права доступу.<br>
3. Створіть клас AccessControl (Контроль доступу) з атрибутами:<br>
   - users (словник, де ключами є імена користувачів, а значеннями - об'єкти класів користувачів).<br>
   - Метод add_user(user), який додає нового користувача до системи.<br>
   - Метод authenticate_user(username, password), який перевіряє, чи існує користувач з таким ім'ям та чи правильний введений пароль. Повертає об'єкт користувача у разі успішної аутентифікації, і None у разі невдачі.<br>
